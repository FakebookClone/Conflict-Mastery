Step 1 - Pull from Master Branch to get any updates

Step 2 - Create a new branch to start working on a single feature

Step 3 - Work on your feature on the new branch and then add to staging area after you're finsihed

Step 4 - Then in the console do a git add . to stage the repo.

Step 5 - Git commit -m "detailed message here"

Step 6 - Git checkout master

Step 7 - git pull ("fast-forward" means no conflicts)

Step 8 - git checkout branchname

Step 9 - git merge master (this will merge master into your branch)

Step 10 - git log (if needed. It shows recent commits)

Step 11 - NEVER git push origin master. We don't work off master!

Step 12 - git push origin branchname (If you git push origin ranchname -u, you can simply git push in the future for that branch

Step 13 - if there are merge conflicts, you need a special git commit after resolving them.
