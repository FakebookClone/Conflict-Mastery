NOTE: Brandon -- Forks are for eating, not git hubbing with teams

Step 1 - Switch to and Pull from Master Branch to get any updates

Step 2 - Switch to your working branch, or create a new branch to start working on a single feature

Step 3 - Work on your feature on the new branch and then add/commit after you're finished.

Step 4 - Checkout to master

Step 5 - git pull to make sure your master is up to date

NOTE: If you take more than 3 seconds, be sure to update master by switching to master and pulling it again. master changes quickly.

Step 6 - Switch back to your working branch, do your work

Step 7 - Then in the console do a git add . to stage the repo.

Step 8 - Git commit -m "detailed message here"

Step 9 - Git checkout master

Step 10 - git pull ("fast-forward" means no conflicts)

Step 11 - git checkout branchname

Step 12 - git merge master (this will merge master into your branch)

NOTE: git log (if needed. It shows recent commits)

NOTE: NEVER git push origin master. We don't work off master!

Step 13 - git push origin branchname (If you git push origin branchname -u, you can simply git push in the future for that branch

NOTE: if there are merge conflicts, you need a extra git commit after resolving them.

Step 14 - make an offical git hub pull request to merge your completed branch to master once and for all

Step 15 - after pull request is merged to master, the one who accepted the request will delete your branch

Step 16 - delete (git branch -d branchname) your feature branch off of your local machine after it is merged to master
